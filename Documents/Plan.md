# План автоматизации тестирования сервиса покупки туров "Путешествие дня"

## 1. Введение

Данный документ описывает план автоматизации тестирования веб-сервиса "Путешествие дня", который предлагает покупку
туров двумя способами:

- Обычная оплата по дебетовой карте
- Выдача кредита по данным банковской карты

Приложение взаимодействует с двумя банковскими сервисами (Payment Gate и Credit Gate) через эмулятор, а также сохраняет
информацию о платежах в СУБД (MySQL и PostgreSQL).

## 4. Перечень и описание возможных рисков при автоматизации

| №  | Риск                                     | Вероятность | Влияние | Способ снижения                                                                      |
|----|------------------------------------------|-------------|---------|--------------------------------------------------------------------------------------|
| 1  | **Нестабильность SUT**                   | Средняя     | Высокое | Добавить ожидания и таймауты в тестах (Selenide), ретраи при падениях                |
| 2  | **Проблемы с Docker**                    | Низкая      | Высокое | Использовать healthcheck для проверки готовности сервисов, добавить sleep в скриптах |
| 3  | **Проблемы с ChromeDriver**              | Средняя     | Среднее | Использовать WebDriverManager для автоматической установки драйверов                 |
| 4  | **Несовместимость Java 11 с Selenide 7** | Средняя     | Среднее | Использовать Selenide 6.19.1, совместимый с Java 11                                  |
| 5  | **Проблемы с application.properties**    | Низкая      | Высокое | Правильное монтирование файла в Docker, дублирование настроек в environment          |
| 6  | **Сложности с проверкой БД**             | Низкая      | Среднее | Использовать Apache DBUtils, писать отдельные методы для каждой таблицы              |
| 7  | **Нестабильность веб-элементов**         | Средняя     | Среднее | Использовать надежные CSS-селекторы, Selenide с умными ожиданиями                    |
| 8  | **Время выполнения тестов**              | Средняя     | Низкое  | Запуск тестов параллельно (Gradle), использование headless режима                    |
| 9  | **Зависимость от внешних сервисов**      | Низкая      | Высокое | Gate Simulator запускается локально, независимость от сети                           |
| 10 | **Неожиданные баги в приложении**        | Высокая     | Среднее | Это цель проекта - найти баги и задокументировать их в Issues                        |
