# Отчет о проведенном тестировании

## 1. Краткое описание

Проведено автоматизированное тестирование веб-сервиса **"Путешествие дня"**, который предоставляет возможность покупки туров двумя способами:
- **Оплата по дебетовой карте** (Payment Gate)
- **Оплата в кредит** (Credit Gate)

Тестирование охватывало:
- UI-сценарии (позитивные и негативные)
- Проверку корректности сохранения данных в СУБД (MySQL)
- Взаимодействие с эмулятором банковских сервисов (Gate Simulator)

Тестирование проводилось на окружении, развернутом с использованием Docker Compose (MySQL, PostgreSQL, Gate Simulator, SUT).

**Окружение:** Windows 11, Chrome 149, Java 17, Gradle 8.5, Selenide 6.19.1

## 5. Общие рекомендации

На основе проведенного тестирования рекомендуется:

### 5.1. Критические исправления
1. **Исправить сохранение платежей в БД** — платежи не сохраняются или сохраняются с некорректным статусом
2. **Исправить отображение уведомлений об ошибках** — для DECLINED карт уведомление не появляется

### 5.2. Валидация полей
3. **Исправить сообщения об ошибках** — заменить "Неверный формат" на конкретные сообщения:
    - "Номер карты должен содержать 16 цифр"
    - "Поле обязательно для заполнения"
    - "Неверный месяц"
    - "Срок действия карты истек"
    - "CVV должен содержать 3 цифры"
    - "Имя должно содержать только буквы"

### 5.3. Валидация даты
4. **Исправить валидацию месяца** — месяц 13 должен давать ошибку "Неверный месяц", а не "Истёк срок действия карты"
5. **Добавить валидацию истекшего срока** — карты с годом меньше текущего должны отклоняться

### 5.4. Валидация CVV
6. **Добавить валидацию CVV** — принимать только 3 цифры

### 5.5. Валидация имени владельца
7. **Добавить валидацию имени владельца** — принимать только буквы и пробелы

### 5.6. Общие улучшения
8. **Унифицировать сообщения об ошибках** — единый стиль и формат
9. **Добавить обработку краевых случаев** — граничные значения для всех полей
10. **Проверить работу с PostgreSQL** — тесты проводились только с MySQL
